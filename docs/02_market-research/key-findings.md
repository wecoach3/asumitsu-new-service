# リサーチ主要発見まとめ

このドキュメントは、アスミツ新規サービス（履歴書×スカウト）の企画にあたり、グローバル＆日本のキャリア系サービスを徹底リサーチした結果の主要発見をまとめたものです。

---

## 発見1: 海外の履歴書サービスは参考にならない

### なぜこの結論に至ったか

海外のResume Builder（Resume.io、Zety、Rezi等）を12サービス詳細に調査した。いずれも「AIで文章を最適化する」「デザインで差別化する」「ATSスコアを上げる」といった機能で競争している。

しかし、**日本の履歴書はJIS規格でフォーマットが決まっている**。名前、写真、学歴、職歴、志望動機——書く欄が固定されていて、デザインの自由度がほぼない。つまり海外サービスの「どう書くか・どう見せるかで差がつく」という前提自体が、日本では成り立たない。

### 具体的に何が使えないか

| 海外で流行っている機能 | 日本で使えない理由 |
|----------------------|------------------|
| AIで文章を最適化 | フォーマットが決まっているので最適化の余地がほとんどない |
| ATSスコアリング（書類通過率を点数化） | 年収300万台の求人を出す中小企業はATS（自動選考システム）を導入していない |
| デザインテンプレートの差別化 | JIS規格で見た目が統一されている。凝ったデザインにする文化がない |
| フリーミアム個人課金（月$20〜30） | そもそも個人課金なしの方針だが、日本の履歴書は穴埋めなので課金するほどの付加価値を出しにくい |

### ただしビジネスモデルや集客手法は参考になる

海外サービスの「機能」は使えないが、「どうやって儲けているか」「どうやってユーザーを集めているか」は参考になる。

| 参考にすべき点 | どのサービスから | 具体的に何が使えるか |
|--------------|----------------|-------------------|
| 無料ツール→B2Bスカウトの構造 | yagish（日本）, Indeed | ユーザーに無料でツールを提供し、集まったユーザーデータを企業に売る。私たちの方針に合致 |
| チャット×モバイルファーストのUX | Hirect | LINEライクなUIで企業と直接チャット。年収300万台のスマホネイティブ層に最適 |
| 「受かった人のサンプル」コンテンツ | Kickresume | 「高卒・工場3年の人がどう書いて受かったか」を見せるコンテンツ。SNSでもSEOでも集客力がある |
| 職種別SEOコンテンツ→ツール誘導 | Zety | 「工場勤務 履歴書 書き方」等の記事を量産し、検索流入からツールへ誘導。SNSと並ぶ第2の集客チャネルになりうる |
| 企業側も基本無料で参入障壁を下げる | Wellfound | 求人掲載を無料にすることで企業を集めやすくする。ユーザー数が少ない初期に有効 |

## 発見2: yagishモデルが最も参考になる

### なぜこの結論に至ったか

日本で「無料の履歴書ツール→B2Bで収益化」を実現しているのはyagishだけ。リクナビNEXTやdodaも履歴書ツールを提供しているが、あくまで自社転職サイトの付随機能であり、履歴書ツール自体でビジネスを作っているわけではない。yagishは「履歴書ツール単体」でビジネスを成立させた唯一の事例。

### yagishの核心（年商5億円の仕組み）

yagishは2018年にサービス開始、2025年で年商約5億円（TBS「がっちりマンデー」で公表）。

ビジネスの本質はシンプル：**「履歴書を今まさに作っている人」は、最も転職意欲が高い層**。この層のデータを広告費ゼロ（100%SEO自然流入）で集め、企業に月1万円で売っている。

| 指標 | 数値 |
|------|------|
| 累計利用者 | 1,000万人 |
| 登録会員 | 250万人 |
| ヤギオファー導入企業 | 6,000社 |
| 企業課金 | 月額1万円（スカウト100通込み） |
| 集客コスト（CAC） | ほぼゼロ（SEO自然流入のみ、広告なし） |
| 年商 | 約5億円 |

### 私たちが同じ構造を使える理由

yagishと全く同じことをやっても勝てない。だが、yagishの**構造**（無料ツール→ユーザー集めて→企業にスカウト機能を売る）は、集客チャネルを変えれば再現できる。

- yagishの集客は**SEO一本**。「履歴書 作成」「職務経歴書 テンプレート」等の検索流入のみ
- **yagishはSNSを全くやっていない**。TikTok、Instagram、Threadsでの発信がない
- つまり**SNS経由で「履歴書を作りたい」と思う層は、今誰も取っていない**
- 私たちの武器はSNSマーケ。適職診断でCPA300円・35,000人のLINE登録を実現済み

### yagishの弱点（私たちが攻めるべきポイント）

yagishのApp Storeレビューや口コミを調査した結果、以下の弱点が見えた。

| yagishの弱点 | 具体的な問題 | 私たちの攻め方 |
|-------------|------------|--------------|
| **SEO一本足打法** | Googleのアルゴリズム変更で一気に崩壊するリスク。実際に検索順位が下がれば集客が止まる | SNSマーケが得意。SEOに依存しない集客基盤を持てる |
| **プライバシー問題** | App Store低評価の主因。「登録後にエージェントから大量の電話勧誘が来る」「個人情報を抜かれた」という不満が多数 | ユーザーが自分で公開範囲を選べる透明な設計にする。yagishの最大の不信感を逆手に取る |
| **C向け課金モデルが苦戦** | 月額550円のyagiprime（AI志望動機生成等）を2025年11月に一部終了。個人課金が軌道に乗っていないことを示唆 | 最初から個人課金なし方針。ユーザーにとっての完全無料を徹底する |
| **履歴書作成で終わり** | 履歴書を作った「その先」（どの求人に応募するか、面接対策等）をカバーしていない | 送客やスカウトで「その先」まで繋げることで、ユーザー体験を完結させる |
| **SNS不在** | SNSプロモーションを一切やっていない | TikTok/Instagram/Threadsで集客。yagishが取れていない層を獲得できる |

## 発見3: 年収300万台のスカウト市場は完全な空白

### なぜこの結論に至ったか

日本のスカウト型（ダイレクトリクルーティング）サービスを10社以上調査した結果、**年収300〜400万円層を対象にしたスカウトサービスは1社も存在しない**ことがわかった。

### 各サービスのターゲット年収帯

| サービス | ターゲット年収 | 備考 |
|----------|-------------|------|
| ビズリーチ | 750万円以上 | 日本最大のハイクラス転職 |
| リクルートダイレクトスカウト | 600万円以上 | ヘッドハンター4,300名 |
| doda X | 600万円以上 | 非公開求人中心 |
| AMBI | 400〜800万円 | 若手ハイキャリア |
| Green | IT人材（年収帯は幅広い） | エンジニア・クリエイティブ52% |
| 転職ドラフト | エンジニア（400万円〜） | 審査合格率40%の厳選制 |
| ミイダス | 幅広い（唯一低年収もカバー） | ただしメインターゲットではない |
| **年収300万台向け** | **存在しない** | — |

### なぜ空白なのか（構造的理由）

スカウトサービスの多くは**成功報酬モデル**（採用成立時に年収の15〜20%を企業が支払う）で収益を得ている。年収300万円の20%は60万円。ビズリーチの主要顧客（年収750万以上）なら150万円。**同じ工数で稼げる金額が半分以下**なので、全社がハイクラスに逃げている。

ただし、これは成功報酬モデルの問題であって、市場がないわけではない。**月額サブスク型（Wantedlyは月5〜16万円・成功報酬なし）なら、年収帯に関係なくビジネスが成立する。**

### 低年収層向けの既存サービスは全てエージェント型

年収300万台・高卒/専門卒向けには、ハタラクティブ、UZUZ、DYM就職、就職Shop等のサービスが存在する。しかし**全てエージェント型**（人が介在して1対1で支援する）であり、企業から求職者に直接アプローチする**スカウト型は1つもない**。

エージェント型はスケーラビリティに限界がある（UZUZは1人あたり平均12時間サポート）。スカウト型のプラットフォームを作れば、この層に初めてスケーラブルにリーチできる。

## 発見4: 「職務経歴書」と「履歴書」は全く別物

### なぜこれが重要か

海外の「Resume」を調べた時に気づいた。海外のResumeは、日本でいう「職務経歴書」に近い（自由形式で職歴・スキル・実績を記述する書類）。一方、日本の「履歴書」はフォーマットが決まった穴埋め式。この2つは全く性質が違う。

そして、私たちのターゲット層（年収300万台・高卒/専門卒・初めての転職）にとって、この違いは決定的に重要。

### ターゲット層にとっての違い

| | 履歴書 | 職務経歴書 |
|--|--------|-----------|
| 馴染み | ◎ バイトの面接でも書いた経験がある | ✕ 存在すら知らない人も多い |
| 心理的ハードル | 低い（欄を埋めるだけ） | 高い（「自己PR？実績？何を書けば…」） |
| 必要な場面 | バイト、パート、正社員、なんでも | 主に正社員の中途転職 |
| 作れるか | ほぼ全員作れる | 初めての転職だと作り方がわからない |

### 結論：入口は絶対に「履歴書」

- 「履歴書」なら間口が圧倒的に広がる。バイト探す人も、初めての転職の人も、全員来れる
- yagishの利用者構成も「中途転職70%、バイト/パート20%、新卒10%」。バイト/パートの20%は「履歴書」だからこそ取れている層
- 職務経歴書は「必要になった人」に後から提供すればいい。入口で出すとハードルが上がって離脱する
- なお、海外のResume Builderの機能（AI最適化、スコアリング等）が日本で使えないのも、「Resume＝職務経歴書的なもの」であり「日本の履歴書」とは別物だから

## 発見5: SNSとSEOではユーザーの温度感が全く違う

### なぜこれが重要か

yagishと同じ構造（無料ツール→B2B）を使うとしても、**集客チャネルが違えばユーザーの「気持ち」が全く違う**。この違いを無視してyagishと同じツール設計をすると失敗する。

### 具体的な違い

| | yagish（SEO経由） | 私たち（SNS経由） |
|--|-------------------|-----------------|
| ユーザーの行動 | 「履歴書 作成」でGoogle検索 | TikTok/Instaでキャリア系動画を見る |
| 動機 | 今すぐ履歴書が必要 | なんとなく気になる、今の仕事にモヤモヤ |
| 温度感 | 高い（既に転職活動中） | 低い（まだ動いていない） |
| 求めているもの | 効率的に履歴書を作れるツール | 自分のキャリアについて考えるきっかけ |

### だから設計が変わる

yagishは「今すぐ履歴書が必要な人」向けに作られているので、穴埋め式で効率よく作れればそれでいい。

私たちのSNS経由のユーザーは「まだ動いていない人」なので、**「穴埋めしてください」では動かない**。「やってみたら面白そう」「結果を見てみたい」という好奇心をトリガーにする必要がある。

適職診断がまさにこの設計で成功している。「診断やってみよう」→「結果が意外だった」→シェア→バズ。新サービスでも同じ心理構造（好奇心→体験→シェア）を組み込むべき。

## 発見6: LINE登録は必須ステップにしなくていい

### なぜこの結論に至ったか

適職診断では「SNS→診断→結果はLINEで」という導線が機能している。しかし履歴書サービスは性質が違う。

| | 適職診断 | 履歴書サービス |
|--|---------|-------------|
| サービスの性質 | コンテンツ（結果を受け取る） | ツール（作って使う） |
| LINEとの相性 | ◎ 結果をLINEで送るのは自然 | △ 「履歴書作りたいのにまずLINE登録？」は邪魔 |
| 参考事例 | — | yagishもLINE挟んでいない。ブラウザ/アプリで直接作成 |

### 結論

- 履歴書を作りたい人にとって、LINE登録は**余計なステップ**。離脱の原因になる
- サービス上で直接使えるようにする（ブラウザ or アプリ）
- ユーザー情報は**段階的に取得**する設計にする
  - 最初：個人情報なしで体験できる
  - 保存/PDF出力：無料アカウント作成（メールアドレス程度）
  - スカウトを受け取る：経歴を詳しく登録

## 確定した方針

ここまでのリサーチと議論を通じて、以下の方針が確定した。

| 方針 | 理由 |
|------|------|
| 入口は「履歴書」 | ターゲット層（年収300万台・高卒/専門卒）にとって馴染みがあり、心理的ハードルが低い。職務経歴書は知らない人も多い |
| 個人課金なし（完全無料） | yagishのyagiprime（月550円）も苦戦して一部終了している。この層から課金するのは難しい |
| SNSで集客 | yagishはSNS不在。SNS経由の履歴書ユーザーは誰も取っていない。私たちの最大の武器 |
| 初期マネタイズは送客 | エージェント紹介で収益化。適職診断で既に送客の仕組みがある |
| 将来B2B課金 | ユーザー数が増えたら企業向けスカウト機能を開放（yagishのヤギオファーと同じ構造） |
| LINE登録は必須にしない | 履歴書はツール。LINE挟むと離脱する。サービス上で直接使える設計にする |
| 段階的に情報開示 | SNS経由の初見ユーザーに個人情報を求めると「怪しい」と思われる。最初は個人情報なしで使えるようにする |
| プライバシー重視 | yagish最大の弱点（エージェント電話勧誘→不信感）の裏返し。ユーザーが自分で公開範囲を選べる透明な設計にする |
| 構造はyagishと同じ | 無料ツール→ユーザー集めて→送客/B2Bスカウト。証明済みのモデルに乗る |
| コンセプトで差別化 | yagishと同じ「穴埋めツール」ではSNSユーザーは動かない。好奇心をトリガーにするコンセプトが必要 |
