# リサーチ主要発見まとめ

## 発見1: 年収300万台のスカウト市場は完全な空白

- 日本のスカウトサービスは全てハイクラス（年収600万+）に集中
- ミイダスが唯一低年収もカバーするが、メインターゲットではない
- 構造的理由: 成功報酬モデル（年収の15-20%）だと低年収帯ではROIが合わない
- → 月額サブスク型（Wantedly型）なら年収帯に関係なく成立する

## 発見2: yagishモデルが最も参考になる

- C向け無料で1,000万人集客 → B2B「ヤギオファー」で7,900社
- 「無料履歴書ツール→ユーザー集めて→企業にスカウト機能を売る」ハイブリッド
- yagishの集客はSEO中心 → 御社のSNSマーケなら集客チャネルが被らない

## 発見3: 海外の履歴書ツールは直接参考にならない

- 日本の履歴書はJIS規格でフォーマット固定（穴埋め作業）
- 海外Resume Builderの価値（書き方・見せ方の最適化）は日本では薄い
- ただしビジネスモデルとマーケティング手法は参考になる

## 発見4: 参考にすべきビジネスモデル・手法

| 何を | どこから | なぜ |
|------|---------|------|
| 無料ツール→B2Bスカウト | yagish | 御社方針に完全合致 |
| 無料ツール→プラットフォーム囲い込み | Indeed | 送客モデルとの相性 |
| チャット×モバイルファーストのUX | Hirect | 年収300万台・スマホネイティブ層に最適 |
| 「受かった人のサンプル」コンテンツ | Kickresume | SNS・SEO両方で使える |
| 職種別SEOコンテンツ→ツール誘導 | Zety | 第2の集客チャネル |
| 年収先出し・厳選モデル | Hired | 「選ばれた感」の演出 |
| 企業側も基本無料 | Wellfound | 企業の参入障壁を下げる |

## 発見5: 退職代行はユーザー属性が完全一致

- 20代・年収300万台が退職代行のメイン層 = 適職診断ユーザーと同じ
- 退職代行の本質は「人材紹介への送客装置」
- SNSバイラル性が極めて高い
- ただし法的リスクあり（中長期の検討課題）
