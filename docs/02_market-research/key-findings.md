# リサーチ主要発見まとめ

## 発見1: 海外の履歴書サービスは参考にならない

### 根本的な違い
- **海外のResume**: フォーマット自由。デザイン・構成・長さ全て自分次第。「どう書くか」で差がつく
- **日本の履歴書**: JIS規格でフォーマット固定。穴埋め作業。差がつかない

### 海外サービスの主要機能が日本で使えない理由
| 海外の機能 | 日本で使えない理由 |
|-----------|------------------|
| AIで履歴書を最適化 | フォーマットが決まっているので最適化の余地がない |
| ATSスコアリング | 年収300万台の求人を出す中小企業はATS未導入 |
| デザインテンプレートの差別化 | JIS規格で見た目が統一されている |
| フリーミアム個人課金 | 個人課金なし方針 |

### ただしビジネスモデル・手法は参考になる
| 何を | どこから | なぜ |
|------|---------|------|
| 無料ツール→B2Bスカウトの構造 | yagish, Indeed | 御社方針に完全合致 |
| チャット×モバイルファーストのUX | Hirect | 年収300万台・スマホネイティブ層に最適 |
| 「受かった人のサンプル」コンテンツ | Kickresume | SNS・SEO両方で使える |
| 職種別SEOコンテンツ→ツール誘導 | Zety | 第2の集客チャネル |
| 企業側も基本無料で参入障壁を下げる | Wellfound | 企業を集めやすい |

## 発見2: yagishモデルが最も参考になる

### yagishの核心
- 年商約5億円。集客コストほぼゼロ（100%SEO自然流入）
- 累計利用者1,000万人、登録会員250万人、ヤギオファー導入6,000社
- 「履歴書を今作っている人」＝最も転職意欲が高い層のインテントデータをCAC≒0で集め、企業に月1万円で売る

### 御社が使える点
- 「無料履歴書ツール→ユーザー集めて→企業にスカウト機能を売る」ハイブリッドモデル
- yagishの集客はSEO中心 → 御社のSNSマーケなら集客チャネルが被らない
- yagishはSNSを全くやっていない。SNS経由の履歴書ユーザーは誰も取っていない

### yagishの弱点（御社が攻めるべき）
| 弱点 | 御社の武器 |
|------|-----------|
| SEO一本足打法（アルゴリズム変更リスク） | SNSマーケが得意 |
| プライバシー問題（エージェント電話勧誘→低評価の主因） | 透明性で差別化可能 |
| C向け課金モデル苦戦（yagiprime一部終了） | 最初から個人課金なし方針 |
| 履歴書作成止まり（その先がない） | 送客・スカウトで「その先」をカバー |
| SNS不在 | TikTok/Insta/Threadsが得意 |

## 発見3: 年収300万台のスカウト市場は完全な空白

### ポジショニングマップ
- ビズリーチ/doda X/リクルートDS → 年収600万+
- AMBI → 年収400〜800万
- ミイダス → 唯一低年収もカバーするがメインターゲットではない
- **年収300万台向けのスカウト型 → 存在しない**

### 空白の構造的理由
- 成功報酬モデル（年収の15-20%）だと低年収帯ではROIが合わない
- → 月額サブスク型（Wantedly型）なら年収帯に関係なく成立する

### 低年収層向けの既存サービスは全てエージェント型
- ハタラクティブ、UZUZ、DYM就職、就職Shop
- 全て「エージェント型」。企業→求職者の「スカウト型」は不在

## 発見4: 「職務経歴書」と「履歴書」は全く別物

### ターゲット層（年収300万台・高卒/専門卒・初めての転職）にとって
| | 履歴書 | 職務経歴書 |
|--|--------|-----------|
| 馴染み | ◎ バイトでも書いたことある | ✕ 存在すら知らない人も多い |
| 心理的ハードル | 低い（穴埋めするだけ） | 高い（「何を書けばいいの？」） |
| 必要な場面 | バイト〜正社員、なんでも | 正社員の中途転職（主に） |

### 結論
- 入口は**絶対に「履歴書」**。職務経歴書ではない
- 履歴書なら間口が圧倒的に広がる（バイト探す人も、初めての転職の人も全員来れる）
- 海外の「Resume」は日本の「職務経歴書」に近い概念。だから海外サービスの機能が日本の「履歴書」に合わない

## 発見5: SNSとSEOではユーザーの温度感が全く違う

```
yagish（SEO）: 「履歴書 作成」で検索 → 今すぐ必要。転職活動中
御社（SNS）:   TikTokで見かける → なんとなく気になる。まだ動いていない
```

### 設計への示唆
- yagishと同じ「穴埋め履歴書ツール」では、SNSユーザーは動かない
- 「なんとなく層」が触りたくなるコンセプトが必要
- 適職診断はこれに成功している（「やってみたら意外だった」→シェア→バズ）
- 新サービスも同じ心理構造（好奇心→体験→シェア）を設計すべき

---

## 発見6: LINE登録は必須ステップにしなくていい

### 適職診断との違い
- 適職診断: コンテンツ（結果を受け取る）→ LINEが自然な導線
- 履歴書: ツール（作って使う）→ サービス上で完結が自然

### 結論
- 履歴書を作りたい人に「LINE登録してね」は邪魔
- yagishもLINE挟んでいない。ブラウザ/アプリで直接作れる
- 段階的に情報を取得する設計（最初は個人情報なし→使ううちに登録）

---

## 確定した方針

```
✅ 入口は「履歴書」（馴染みがある、ハードル低い）
✅ 個人課金なし（ユーザーは完全無料）
✅ SNSで集客（yagishはSNS不在。ここを取る）
✅ 初期マネタイズは送客（エージェント紹介）
✅ ユーザー数拡大後にB2B課金（企業向けスカウト機能）
✅ LINE登録は必須にしない（サービス上で直接使える）
✅ 段階的に情報開示（最初は個人情報不要）
✅ プライバシー重視（yagishの弱点を突く）
✅ 構造はyagishと同じ（無料ツール→送客→B2Bスカウト）
✅ コンセプトで差別化（yagishと同じ「穴埋めツール」では勝てない）
```
