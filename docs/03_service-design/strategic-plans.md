# アスミツ新規サービス 戦略プラン（5案）

作成日: 2026年4月7日

---

## 背景：なぜ5つのプランを用意するのか

4/11のLINEテスト前に「どの方向に進むのか」の選択肢を整理しておく。テスト結果によってどのプランに進むかを判断する。

### 議論の中で見えた最大の論点

**「AI履歴書」はツールとして完結してしまい、マネタイズに繋がりにくい。**

```
適職診断: 「自分に合う仕事を知りたい」→ 面談（自然）
履歴書:   「書類を作りたい」→ 面談（なんで？）
```

履歴書を作った人に面談させるのは不自然。だからコンセプト（入口の期待値）の設計が全てを決める。

### コンセプトの主従で未来が変わる

```
パターン①: メイン=AI履歴書（ツール） / サブ=求人紹介（出口）
  → ツールとして使われるが、マネタイズが弱い

パターン②: メイン=あなたに合う求人が届く（価値提案） / サブ=履歴書入力（手段）
  → マネタイズが自然。yagishと土俵が違う
```

---

## プラン1：AI履歴書ツール特化型（堅実）

### コンセプト

「AIが履歴書を全部作ってくれる。無料」

- メイン訴求: AI志望動機・自己PR生成
- マネタイズ: 完成後にエージェントアフィリエイト（ポップアップ）
- yagishの無料版ポジションを取りにいく

### 3ヶ月後・半年後・1年後

| 時期 | やること | 売上イメージ |
|------|---------|------------|
| 3ヶ月後 | SNSでツール系コンテンツ投稿。ユーザー増加中 | 月10〜30万 |
| 半年後 | SEOも育ち始める。「アスミツ履歴書」で検索1位 | 月30〜80万 |
| 1年後 | yagishの無料版ポジションを確立 | 月80〜150万 |

### メリット

- リスク低い。ツールとして確実に使われる
- SEOも狙える（「履歴書 作成 無料」等）
- 開発がシンプル

### デメリット

- 面談に繋がりにくい。マネタイズが弱い
- 天井が低い
- yagishと同じ土俵で戦うことになる（SEO勝負では不利）
- スカウトサービスへの発展が遠い

### このプランが正解になる条件

- LINEテストでAI志望動機の利用率が非常に高い
- ツールとしてのバイラル（「これ便利」でシェアされる）が起きる
- マネタイズは後回しでもいいから、まずユーザー数を取りたい場合

---

## プラン2：求人マッチング型（さっきの議論の結論）

### コンセプト

「あなた限定の求人サンプルが届く」

- メイン訴求: 求人が届く驚き。「自分にもスカウトが来るの？」
- 履歴書は求人を届けるための入力手段
- マネタイズ: 最初から送客が自然に組み込まれている

### 3ヶ月後・半年後・1年後

| 時期 | やること | 売上イメージ |
|------|---------|------------|
| 3ヶ月後 | SNSで「高卒の自分にスカウト来た」系コンテンツ。送客開始 | 月30〜80万 |
| 半年後 | 送客安定。訴求パターン確立 | 月80〜200万 |
| 1年後 | Phase 2（企業スカウト）準備開始 | 月200〜300万 + Phase 2仕込み中 |

### メリット

- マネタイズが自然。「求人が欲しい」で来るから面談に繋がる
- yagishと土俵が違う（yagishは「履歴書ツール」、うちは「求人が届くサービス」）
- SNS訴求が強い。「自分にもスカウトが来た」は意外性がありバズりやすい
- 適職診断の成長カーブを再現しやすい

### デメリット

- 「求人が届く」と言って実態はエージェント送客。期待値ギャップのリスク
- yagishのApp Store低評価と同じパターンになる可能性
- 履歴書入力がハードル（求人だけ見たい人が離脱する）

### このプランが正解になる条件

- LINEテストで訴求C（あなた限定の求人サンプル）の遷移率がAより高い
- 送客ポップアップのタップ率が高い
- 電話接続率が適職診断と同等以上

---

## プラン3：適職診断の延長線型（最低リスク）

### コンセプト

適職診断の結果 →「じゃあ履歴書作ろう」→ 求人紹介

- 別アカウントではなく、適職診断のアカウント内で「履歴書も作れるようになった」と案内
- 既存の33,000人のLINEリストに直接アプローチ
- 新規集客の心配がゼロ

### 3ヶ月後・半年後・1年後

| 時期 | やること | 売上イメージ |
|------|---------|------------|
| 3ヶ月後 | 適職診断から履歴書への導線追加。既存リストに案内 | 月+20〜50万（既存に上乗せ） |
| 半年後 | 適職診断→履歴書→送客の一気通貫が回る | 月+50〜100万 |
| 1年後 | マネタイズ率2.88%→5〜8%に改善 | 既存売上が1.5〜2倍に |

### メリット

- 最もリスクが低い。新規集客不要
- 既存の35,349人のLINEリストを直接活用
- 適職診断のマネタイズ率を上げるので、確実にプラス

### デメリット

- 新規サービスではなく、既存サービスの改善にとどまる
- スカウトサービスへの発展性が薄い
- 別アカウントでやると決めた方針と矛盾する
- 天井は適職診断の集客力に完全依存

### このプランが正解になる条件

- 新アカウントのSNS集客がうまくいかなかった場合のフォールバック
- 適職診断のマネタイズ率改善が最優先の場合
- リスクを最小限にしたい場合

---

## プラン4：スカウトプラットフォーム先行型（攻め）

### コンセプト

「年収300万台専門のスカウトサービス」

- 最初からスカウトプラットフォームとして設計
- 「ビズリーチは年収600万から。うちは年収300万から」
- 早期に企業課金（Phase 2）に移行

### 3ヶ月後・半年後・1年後

| 時期 | やること | 売上イメージ |
|------|---------|------------|
| 3ヶ月後 | MVP + 求職者プール開始。送客でマネタイズ検証。**免許申請中** | 月20〜50万 |
| 半年後 | 求職者DB数千人。免許取得完了。企業営業開始 | 月50〜100万 + 企業数社テスト導入 |
| 1年後 | 企業50〜100社導入。月額課金スタート | 月100〜300万（送客）+ 月50〜100万（企業課金） |

### メリット

- 天井が最も高い。企業課金が乗れば爆発する
- CPA300円が最大限活きるビジネスモデル
- ROXXが上場で年収300万台の求職者プールビジネスが成立することを証明済み
- yagishのヤギオファー（6,000社導入）が先行事例

### デメリット

- 免許取得に2〜3ヶ月 + 21万円
- 企業営業が必要（今やっていない領域）
- 立ち上がりが遅い。最初の半年はほぼ投資期間
- やることが多い（サービス開発 + SNS集客 + 企業営業）

### このプランが正解になる条件

- 半年〜1年の投資期間を許容できる
- 企業営業をやる人がいる（or やる覚悟がある）
- 「年収300万台のビズリーチ」というポジションに確信がある

## プラン5：ハイブリッド型（段階的に攻める）

### コンセプト

Phase 1は「求人が届くサービス」で集客・マネタイズ検証 → Phase 2で企業スカウトに移行

- 最初はプラン2と同じ動き
- データが溜まったらプラン4に進化
- 送客 → 企業課金の二段構え

### 3ヶ月後・半年後・1年後

| 時期 | やること | 売上イメージ |
|------|---------|------------|
| 3ヶ月後 | プラン2と同じ。送客でマネタイズ検証。**同時に免許申請開始** | 月30〜80万 |
| 半年後 | 送客安定 + 免許取得完了 + 求職者DB数千人。企業テスト導入開始 | 月80〜200万 + 企業数社 |
| 1年後 | 送客月200〜300万 + 企業課金月100〜200万 | **月300〜500万** |

### Phase移行の判断基準

```
Phase 1 → Phase 2 に移行する条件:
  ✅ 月間アクティブ求職者が1,000人以上
  ✅ 送客でのマネタイズが月100万以上で安定
  ✅ 有料職業紹介の免許取得済み
  ✅ 求職者のプロフィール情報（職種・経験・希望）が蓄積されている
```

### 詳細タイムライン

```
Month 1-3: 求人マッチング型で立ち上げ
  - MVP開発 + LINEテスト + SNS開始
  - 送客でマネタイズ検証
  - 免許申請開始（21万円、2〜3ヶ月）
  - 求職者データの蓄積開始

Month 4-6: 送客を安定させながら、企業側の準備
  - SNS訴求パターン確立
  - 送客月50〜100件を目指す
  - 免許取得完了
  - 企業向けスカウト機能のMVP開発
  - テスト導入企業を3〜5社見つける

Month 7-9: 企業スカウトのテスト開始
  - テスト企業にスカウト機能を無料開放
  - 返信率・採用率のデータを取る
  - 価格設定を検証（月1万？3万？5万？）
  - 送客は継続（月200〜300万）

Month 10-12: 企業課金の本格開始
  - テスト結果をもとに料金確定
  - 企業50〜100社への営業開始
  - 月額課金スタート
  - 送客 + 企業課金の二本柱が成立
```

### メリット

- プラン2で手堅くマネタイズしながら、プラン4の天井の高さも狙える
- リスクを段階的に取る。いつでもプラン2に留まれる
- 免許申請を早めに始めるので、Phase 2への移行がスムーズ
- 適職診断の成長カーブ（初月48万→4ヶ月目400万）を再現しつつ、その先がある

### デメリット

- やることが多い。集中力が分散するリスク
- 「送客も企業課金もどっちつかず」になる可能性
- 企業営業のリソースが必要（Month 4以降）

### このプランが正解になる条件

- プラン2のマネタイズが3ヶ月で月50万以上に達した場合
- 「送客だけでは天井がある」と実感した場合
- 企業営業に割ける人的リソースがある場合

## 5つのプラン比較

| | コンセプト | 3ヶ月後 | 半年後 | 1年後 | リスク | 天井 |
|---|---------|--------|-------|-------|------|------|
| 1 | AI履歴書ツール | 月10〜30万 | 月30〜80万 | 月80〜150万 | 低 | 低 |
| 2 | 求人が届くサービス | 月30〜80万 | 月80〜200万 | 月200〜300万 | 中 | 中 |
| 3 | 適職診断の延長 | +月20〜50万 | +月50〜100万 | 既存1.5〜2倍 | 最低 | 低 |
| 4 | スカウトPF先行 | 月20〜50万 | 月50〜100万 | 月150〜400万 | 高 | 最高 |
| 5 | ハイブリッド | 月30〜80万 | 月80〜200万 | 月300〜500万 | 中 | 高 |

### 4/11 LINEテストの結果でどのプランに進むか

| テスト結果 | 推奨プラン |
|-----------|----------|
| 訴求Aが圧倒的に強い（AI志望動機に価値あり） | プラン1 or プラン3 |
| 訴求Cが強い + 送客タップ率高い | **プラン2 → プラン5** |
| 訴求Cが強い + 送客タップ率低い | プラン2（送客導線を改善） |
| 両方低い | プラン3（適職診断の延長で立て直し） |
| 訴求Cが圧倒的 + 面談も自然に繋がる | **プラン5（一気にハイブリッドへ）** |
