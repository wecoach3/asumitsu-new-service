# アスミツ新規サービス 全体整理

---

## 1. なぜ新規サービスを始めるのか

### 適職診断の現状

アスミツ適職診断はSNS（TikTok/Instagram/Threads）で運営しているキャリア系サービス。SNSで「アスミツ」と検索するよう訴求し、Google検索経由でサービスに流入させるモデル。マネタイズは面談後にエージェントに送客し、成功報酬を受け取る形。

| 指標 | 数値 |
|------|------|
| PV | 158,905 |
| LINE流入 | 35,349（CPA約300円） |
| 面談設定 | 2,352（対LINE 6.65%） |
| 面談実施 | 1,878 |
| エージェント送客 | 1,018（2〜3社に送客、1社あたり約3万円） |
| 年間売上 | 約2,980万円 |

**集客は強い。CPA300円で35,000人。** 適職診断の競合（リクルート、マイナビ、doda、ミイダス等）が大量にいる中でこの数字を出せている。SNSマーケのコンセプトと見せ方が武器。

**しかしマネタイズが弱い。** 35,349人のうちマネタイズできているのは1,018人（2.88%）。93%、約33,000人が寝ている。

### なぜ93%がマネタイズできないのか

**収益導線が「面談→エージェント送客」の1本しかない。** この導線で収益化できるのは「今すぐ転職したい人」だけ。

適職診断をSNSで受ける人は「なんとなくキャリアが気になる」「自分に合う仕事って何だろう」という軽い気持ち。まだ転職する気がない人が大半。だから93%が面談に至らない。

### 潜在層と顕在層

| | 潜在層 | 顕在層 |
|--|--------|--------|
| 状態 | まだ動いていない | もう動いている or 動こうとしている |
| 気持ち | 「なんとなく気になる」 | 「転職活動中」「書類出してる」 |
| エージェント紹介への反応 | 「まだいいかな…」 | 「相談したい」 |

**適職診断は潜在層を集めるサービス。** 潜在層をいくら集めても、マネタイズ率2.88%の構造は変わらない。

**顕在層を取るには、別のサービスが必要。**

### 2つの新規サービス

顕在層を取るために、2つのサービスを立ち上げる。

| | アスミツ履歴書 | スカウトサービス（仮） |
|--|-------------|-------------------|
| 何をするか | AIが志望動機・自己PRを自動生成する無料の履歴書サービス | 履歴書を入力したらスカウト（求人紹介）が届くサービス |
| 獲得する層 | 転職活動中 or これから転職する人 | 同上 |
| マネタイズ | エージェント送客 | エージェント送客 → 将来は企業課金 |

この2つは別々のサービスではなく、**1つのサービスの中の2つの価値**として提供できる可能性がある。詳細は後述。

## 2. 私たちが獲得する層

### 顕在層の中で、私たちがリーチできるのは誰か

適職診断で実績がある層がそのまま対象：

- 年収300万円台
- 高卒・専門卒
- 20代〜30代前半
- SNSが情報源（TikTok/Instagram/Threads）
- 工場、販売、飲食、介護、事務、コールセンター、ドライバー等

### この層の転職活動の特徴

大卒ホワイトカラーとは全く違う世界。

| | 大卒ホワイトカラー | この層 |
|--|------------------|-------|
| 転職期間 | 3〜6ヶ月 | 2週間〜1ヶ月 |
| 選考 | 書類→面接複数回 | 書類→面接1回→内定 |
| 重視されること | スキル・実績 | 人柄・やる気・すぐ来れるか |
| 職務経歴書 | 必須 | **不要な求人が多い** |
| 志望動機 | しっかり書く | **空欄でも受かることがある** |
| デバイス | PC | **スマホだけ** |
| 情報源 | 転職サイト・エージェント | **SNS** |
| エージェント | 普通に使う | **相手にされない感覚がある** |

**日本の給与所得者の中で年収300〜400万は最大のボリュームゾーン（826万人）。** 年間200万人以上が転職している巨大市場。にもかかわらず、この層向けのスカウトサービスは存在しない。

### この層がつまずいていること

#### つまずき①：これから転職する人 —「履歴書が書けない・めんどくさい」

- 志望動機・自己PRを真剣に書いた経験がない
- ネットの例文は大卒ハイスペ向けで参考にならない
- 周りに添削してくれる人がいない
- そもそも履歴書を作ること自体がだるい
- 結果、転職したい気持ちはあるのに動けない

#### つまずき②：転職活動中の人 —「書類が全然通らない」

- 何社出しても書類で落ちる
- 原因が志望動機・自己PRの書き方にあることに気づいていない
- 「学歴が低いから」「経験が足りないから」と思っている

## 3. アスミツ履歴書のコンセプト

### 一言で

「履歴書の書き方がわからない人に、AIが志望動機・自己PRを作ってくれる。無料。」

### サービスの中身

| 機能 | 内容 |
|------|------|
| 履歴書作成 | JIS規格の穴埋め式。スマホ対応 |
| AI志望動機生成 | 職種・経歴を入力→AIが3パターン提案→選ぶだけ |
| AI自己PR生成 | 同上 |
| PDF出力 | 完成した履歴書をPDFでダウンロード |
| 料金 | **全機能完全無料** |

AI生成コストは月10万件で6,000〜8,000円。無料提供しても全く問題ない。

### yagishとの差別化

| | yagish | アスミツ履歴書 |
|--|--------|-------------|
| 訴求 | 「楽に作れる」 | 「AIが書いてくれる。書類通る」 |
| 集客 | SEO（SNSは一切やっていない） | SNS |
| AI志望動機 | 月550円（苦戦して一部終了） | **完全無料** |
| プライバシー | 不透明（エージェント電話勧誘→不信感） | ユーザーが選べる透明設計 |

### マネタイズ

適職診断と同じ導線：

```
履歴書完成 →「求人サンプル見てみますか？」→ キャリア相談 → エージェント送客
```

1人あたり2〜3社に送客、1社あたり約3万円。CPA300円に対してユニットエコノミクスは成立する。

### 検証すべき仮説

**仮説A: この層は志望動機が書けなくて困っている → AI志望動機に価値がある**
**仮説B: この層は履歴書を作ること自体がだるい → 楽さ・速さに価値がある**

この層の求人は志望動機が空欄でも受かることがある。職務経歴書も不要。つまり**AI志望動機の価値が想定より低い可能性がある。** LINEテストで検証する。

## 4. スカウトサービスのコンセプト

### なぜスカウトサービスが成り立つのか

年収300万台向けのスカウトサービスは日本に存在しない。理由は「ユーザーを集めるコストが高い」から。

| サービス | ユーザー1人の獲得コスト（アフィリエイト報酬ベース） |
|---------|-----------------------------------------------|
| doda X | 9,581〜41,825円 |
| ビズリーチ | 6,000〜25,000円 |
| リクルートダイレクトスカウト | 3,648〜35,000円 |
| **私たち** | **300円** |

**桁が違う。** 他社が1人集めるのに1〜4万円かかるところを、私たちは300円でできる。

これが年収300万台向けスカウトサービスが成り立つ根拠。他社は集客コストが高いから企業への課金も高くなり、年収300万台では採算が合わない。**私たちはCPA300円だから、企業への課金を安くしても利益が出る。**

yagishのヤギオファー（月1万円）が成立しているのも、SEOで集客コストがゼロだから。私たちはSNSでCPA300円。同じ構造が使える。

### スカウトサービスの位置づけ

Phase 1（今）: 履歴書サービスでユーザーを集めて、エージェント送客でマネタイズ
Phase 2（将来）: ユーザーが集まったら、企業にスカウト機能を開放して課金

ただし、**Phase 1の段階でも「スカウト訴求」はテストできる**。「履歴書を入れたらあなたに合う求人を紹介します」という形で、実態はエージェント送客だが、ユーザーからすると「求人が届く」体験になる。

## 5. 訴求の設計

### 3つの訴求パターン

| パターン | 訴求 | 何を検証できるか |
|---------|------|----------------|
| A | **「AIが志望動機書いてくれる」** | AI機能に価値があるか（仮説A） |
| B | **「スマホで3分で履歴書完成」** | 楽さ・速さに価値があるか（仮説B） |
| C | **「履歴書入れるだけであなたに合う求人を紹介」** | スカウト訴求が刺さるか + マネタイズが自然に繋がるか |

### パターンCが重要な理由

パターンCは他の2つと根本的に違う。AとBは「履歴書を作る」がゴール。Cは「求人が届く」がゴール。

```
A/Bの導線:
  「履歴書作ろう」→ 作る → 完成 → ポップアップ「求人見る？」→ タップ → 送客
  問題: 「履歴書作りたかっただけなのに電話来た」

Cの導線:
  「求人届くよ」→ 履歴書入れる → 求人紹介が届く → 電話
  違い: 最初から「連絡が来る」ことを期待している → 電話に出る
```

**Cはマネタイズのリスク（電話に出ない問題）を根本から解決する可能性がある。**

### 訴求の具体例

**パターンA（AI志望動機）:**
- 「履歴書の書き方わからない人、AIが全部作ってくれるよ」
- 「高卒で自己PR何も思いつかなかったけど、AIが出してきた文章すごかった」

**パターンB（楽さ・速さ）:**
- 「スマホで3分で履歴書完成した。しかも無料」
- 「もう自分で志望動機書かなくていい時代」

**パターンC（求人紹介/スカウト）:**
- 「履歴書入れただけで自分に合う求人紹介してもらえた」
- 「このサービス使ったおかげで2週間で内定出た」

## 6. 4/11 LINEテストの設計

### 目的

1. **どの訴求が刺さるか**（遷移率で判断）
2. **マネタイズが成立するか**（送客→電話→面談が動くか）

### 最大の構造的リスク

yagishはSEOで毎日12,000人が勝手に来る。A/Bテストがタダで無限にできる。

私たちはSNSで1本1本コンテンツを作って投稿する。バズらなければ人が来ない。人が来なければテストもできない。**改善サイクルがyagishより圧倒的に遅い。**

**だからLINEリスト33,000人が生命線。** 確実に届くLINEで先にデータを取って、SNSを始める前に「何が刺さるか」の当たりをつける。一気に使い切らず、セグメントに分けて段階的にテストする。

### テスト設計

**4/11: 3,500人を1,750人ずつに分割**

| グループ | 人数 | 配信文言 | 検証すること |
|---------|------|---------|------------|
| A | 1,750人 | 「AIが志望動機書いてくれる」 | AI機能に価値があるか |
| C | 1,750人 | 「履歴書入れたらあなたに合う求人を紹介」 | スカウト訴求が刺さるか + マネタイズが自然か |

**なぜAとCか：** BはAの中で体験としてわかる（使ってみたら早かった、となる）。最も知りたいのは「マネタイズできるか」なので、マネタイズへの繋がり方が全く違うAとCを比較する。

### 計測する指標

**コンセプトの検証：**

| 指標 | 何がわかるか |
|------|------------|
| 開封率 | 「AI履歴書」「求人紹介」に興味があるか |
| サービスへの遷移率 | 「使ってみたい」と思う人の割合。**AとCでどちらが高いか** |
| 履歴書作成の完了率 | 最後まで使えるか |
| AI志望動機の利用率 | AI機能に価値を感じるか（仮説A/Bの判断材料） |

**マネタイズの検証：**

| 指標 | 何がわかるか |
|------|------------|
| 送客ポップアップのタップ率 | 「求人サンプル見る？」に興味あるか |
| 電話の接続率 | 繋がるか。**AとCで差が出るか** |
| キャリア相談への転換 | 面談に至るか |

### テスト後の判断

| 結果 | 次のアクション |
|------|-------------|
| Aの遷移率が高い | 「AI志望動機」をメイン訴求にしてSNS開始 |
| Cの遷移率が高い | 「求人紹介」をメイン訴求にしてSNS開始。スカウトサービスの優先度を上げる |
| 両方低い | 配信文言を変えて2回目のテスト（別の3,500人） |
| Cの方が電話接続率が高い | スカウト訴求はマネタイズに直結する。Phase 2を前倒し検討 |

## 8. リスクまとめ

### 構造的リスク

| リスク | 詳細 | 対策 |
|--------|------|------|
| **テストサイクルが遅い** | yagishは毎日12,000人でA/Bテストし放題。うちはSNSなのでバズらないとデータが集まらない | LINEリスト33,000人を段階的にテストに使う |

### コンセプトのリスク

| リスク | 詳細 | 対策 |
|--------|------|------|
| この層は志望動機を空欄で出している | 飲食・工場・介護は志望動機なしでも受かる。AI志望動機の価値が低い可能性 | LINEテストでAI利用率を見て判断。低ければ「楽さ」訴求にシフト |
| 職務経歴書がそもそも不要 | この層の求人は履歴書だけで完結することが多い | 入口は「履歴書」に絞る（既にそうしている） |
| SNSで「履歴書」は重すぎる | 適職診断の「やってみよう」と比べて、「履歴書作ろう」は重い行動 | パターンCの「求人が届く」訴求なら入口が軽くなる可能性 |

### マネタイズのリスク

| リスク | 詳細 | 対策 |
|--------|------|------|
| **電話に出ない** | この層は知らない番号に出ない。「タップしただけで電話来ると思ってない」 | パターンC（最初から連絡が来ると期待させる）で解決の可能性。LINE/チャットでの面談設定も検討 |
| 「履歴書作っただけなのにキャリア相談？」 | 適職診断からの人とは動機が違う | 「書類通過のコツ教えます」等、履歴書の延長線上の切り口で相談に繋ぐ |
| 送客先エージェントが合わない | 年収300万台・高卒向けのエージェントが必要 | ハタラクティブ、UZUZ等に切り替え |
| 1ヶ月後に「ユーザーは来てるが1円にもなってない」 | 最悪のシナリオ | 送客導線の各ステップを分解して、どこでコケているか特定→そこを改善 |

### 競合リスク

| リスク | 詳細 | 対策 |
|--------|------|------|
| yagishがAI志望動機を無料化 | 私たちを見て追随する可能性 | yagishはSEO集客。SNSとは被らない。先にSNSでポジションを取る |
| 同じコンセプトのスタートアップ | AI履歴書は参入障壁が低い | 先行者優位。CPA300円の集客力は簡単に真似できない |

## 9. なぜ私たちがやるべきなのか（まとめ）

**3つの武器がある：**

1. **CPA300円のSNS集客力。** 他社がユーザー1人に1〜4万円かけているところを300円でできる。これが全ての基盤
2. **33,000人のLINEリスト。** テスト基盤として使える。新サービスの初速を出せる
3. **エージェント送客の既存導線。** 面談→送客の仕組みは適職診断で1,878人面談・1,018人送客の実績がある

この3つがあるから：
- **履歴書サービス**で顕在層を獲得し、送客でマネタイズできる
- **スカウトサービス**は、他社では採算が合わない年収300万台でも成立する
- **適職診断（潜在層）+ 新サービス（顕在層）**でキャリア領域を全カバーできる
