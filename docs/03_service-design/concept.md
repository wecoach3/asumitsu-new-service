# サービスコンセプト

---

## 一言で

「履歴書の書き方がわからない人に、AIが志望動機・自己PRを作ってくれる。無料。」

---

## 背景：なぜこのサービスをやるのか

### アスミツ適職診断の現状

アスミツ適職診断はSNS（TikTok/Instagram/Threads）からの集客でLINE登録35,349人を獲得している。しかしマネタイズできているのは全体の2.88%（1,018人）のみ。93%のリストが「寝ている」状態。

原因は明確で、収益導線が「面談→エージェント送客」の1本しかなく、「今すぐ転職したい人」しか収益化できていない。

### 適職診断が取っている層と、取れていない層

```
適職診断が取れている層:
「自分に合う仕事って何だろう？」（潜在層・まだ動いていない人）

取れていない層:
「転職活動中だけど書類で苦戦している」（顕在層・もう動いている人）
```

新サービスは**この「取れていない層」を獲得する**ためのもの。適職診断と補完関係にあり、カニバらない。

## ターゲット

**転職活動中、またはこれから転職しようとしている人。**

具体的には：
- 年収300万円台
- 高卒・専門卒
- 20代〜30代前半
- 履歴書の書き方に自信がない（特に志望動機・自己PR）
- 初めての転職 or 転職経験が少ない

### このターゲットの心理

「めんどくさい」の前に、**「そもそもどう書けばいいかわからない」**がある。

```
× 私たちの最初の想定: 「志望動機書くのめんどくさい」→ AIが代わりにやる
○ 実際のユーザー心理:  「志望動機？自己PR？何書けばいいの？」→ 不安 → 自信がない → 動けない
```

履歴書は「めんどくさいもの」ではなく「怖いもの」に近い。書き方がわからなくて不安だから動けない。

## 訴求（SNSで何を言うか）

### ユーザー心理に沿った3段階

| 段階 | ユーザーの心理 | 訴求の例 |
|------|-------------|---------|
| **入口** | 書き方わからない、自信ない | 「履歴書の書き方わからない人、AIが全部作ってくれるよ」 |
| **体験** | 使ってみたらできた | 「自己PRも志望動機もAIが出してくれた。しかも無料」 |
| **成果** | 結果が出た | 「これで書類通った」「書類の通過率上がった」 |

### SNSコンテンツの基本フォーマット

```
「正直、履歴書の書き方全然わからんかった」
 ↓
「でもこのアプリ使ったらAIが志望動機も自己PRも全部出してくれた」
 ↓
「しかも無料」
 ↓
「これで出したら普通に書類通ったｗ」
```

入口（共感）→ 体験（解決）→ 成果（結果）のストーリーを1本の動画にする。

### 訴求のバリエーション

**「わからない」を解決する系:**
- 「履歴書の書き方わからんかったけど、AIが全部教えてくれた」
- 「自己PRとか何書けばいいか意味不明だったけど、AIに聞いたら3秒で出てきた」
- 「志望動機の書き方ググるのやめて、AIに作ってもらった方が早かった」

**「自信ない」を解決する系:**
- 「高卒で自己PR何も思いつかなかったけど、AIが出してきた文章が普通にすごかった」
- 「自己PR書く自信なかったけど、AIが自分の経歴からいい感じに作ってくれた」

**成果系:**
- 「これ使ったら書類通過率が上がった。だからみんな使った方がいいよ」
- 「書類で落ちまくってたけどこれ使ってから通るようになった」

**スピード系:**
- 「志望動機も自己PRも30秒で終わった」
- 「もう自分で志望動機書かなくていい時代」

## サービスの中身

### 基本機能
- 履歴書作成（JIS規格の穴埋め式）
- **AIが志望動機を自動生成**（職種・経歴を入力するだけ）
- **AIが自己PRを自動生成**（同上）
- PDF保存 / コンビニ印刷
- 全機能完全無料

### ユーザー体験フロー

```
1. SNSで訴求を見る（「履歴書の書き方わからない人、AIが作ってくれるよ」）
2. サービスにアクセス（ブラウザ or アプリ）
3. 基本情報を穴埋め（名前・学歴・職歴）
4. 志望動機 → AIが3パターン提案 → 選ぶだけ
5. 自己PR → AIが3パターン提案 → 選ぶだけ
6. 完成 → PDF保存 / コンビニ印刷
7. 「この経歴で受けられる求人を見る？」→ 送客
```

### LINE登録は必須にしない

適職診断は「SNS→LINE登録→結果受け取り」の導線だが、履歴書サービスはツールなので**サービス上で直接使える**設計にする。LINE挟むと離脱する。

### 段階的な情報取得

SNS経由の初見ユーザーに個人情報をいきなり求めると「怪しい」と思われる。

```
Step 1: 個人情報なしで体験できる（AIの志望動機生成を試す等）
Step 2: 保存/PDF出力するなら無料アカウント作成（メールアドレス程度）
Step 3: スカウトを受け取りたいなら経歴を詳しく登録
```

## yagishとの差別化

### ポジショニングの違い

| | yagish | 私たち |
|--|--------|--------|
| **ユーザーの来る理由** | 「履歴書が必要だから」（ツール目的） | 「書き方わからないから助けてほしい」（不安の解決） |
| **訴求** | 「楽に作れる」（利便性） | 「AIが書いてくれる。書類通る」（不安の解消＋成果） |
| **集客** | SEO（100%検索流入） | SNS（TikTok/Instagram/Threads） |
| **AI志望動機** | 月550円（苦戦して一部終了） | **完全無料** |
| **プライバシー** | 不透明（エージェント電話勧誘→不信感） | ユーザーが公開範囲を選べる透明設計 |

### 既存のAI志望動機サービスとの違い

マイナビ、doda、リクナビも既にAI志望動機生成を無料で提供しているが、**全て自社転職サイトの会員限定機能**。

| | 大手転職サイト | 私たち |
|--|-------------|--------|
| 利用条件 | マイナビ/doda等に会員登録が必要 | **会員登録なしでも使える** |
| 位置づけ | 転職サイトの付随機能 | **独立したサービス** |
| 集客 | 自社サイトの会員向け | **SNSで直接バズらせる** |

yagishが「リクナビもdodaも履歴書ツールを持っていたけど、独立した履歴書サービスとして出して1,000万人集めた」のと同じ構造。

AI志望動機も「大手の転職サイト内機能」ではなく「独立したサービス」として出すことで、同じことが起きる可能性がある。

## マネタイズ

### Phase 1（初期）: 送客

履歴書を作った人は転職活動中。書類はできた、次は応募先。ここでエージェントを紹介する。

yagishとの決定的な違い：**私たちのユーザーの方が送客の転換率が高いはず。**

```
yagishのユーザー:
「履歴書作って応募するだけ」→ 応募先は既に決まっている → スカウトもエージェントも「間に合ってます」

私たちのユーザー:
「書類で苦戦中。助けが欲しい」→ まだ転職先が決まっていない → エージェント紹介もスカウトも受け入れやすい
```

### Phase 2（ユーザー数拡大後）: B2Bスカウト

ユーザーが集まったら、企業向けにスカウト機能を開放する。yagishのヤギオファー（月額1万円・6,000社導入）と同じ構造。

## 適職診断との関係

カニバらない。補完関係。

| | 適職診断 | 新サービス |
|--|---------|-----------|
| 取る層 | 潜在層（まだ動いてない人） | 顕在層（もう動いてる or 動こうとしている人） |
| 入口の動機 | 「自分に合う仕事って何？」 | 「履歴書どう書くの？」 |
| ユーザーの温度感 | 低い | 高い |
| 送客転換率 | 6.65%（LINE→面談設定） | これより高くなるはず |
| 集客チャネル | SNS → Google検索「アスミツ」 | SNS → 直接サービスへ |

**両方持つことで、キャリア領域の潜在層〜顕在層を全カバーできる。**

## 技術・コストの実現性

### AI生成コスト

| モデル | 志望動機1件あたり | 月10万件 |
|--------|----------------|---------|
| Gemini 2.0 Flash | 約0.06円 | 約6,000円 |
| GPT-4o-mini | 約0.08円 | 約8,000円 |
| GPT-4o | 約1.4円 | 約14万円 |

**月10万件生成しても6,000〜8,000円。** 完全無料で提供してもコスト的に全く問題ない。

### 技術的な実現性

- LLM APIを呼ぶだけで基本的に動く。開発コストは低い
- 職種・経歴を入力→志望動機を出力するプロンプト設計は確立されている
- 既に複数の日本サービスが実現済みで、技術的なリスクはほぼない
