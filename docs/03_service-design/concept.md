# アスミツ履歴書 サービスコンセプト

---

## サービス名（仮）

**アスミツ履歴書**

---

## 一言で

「履歴書の書き方がわからない人に、AIが志望動機・自己PRを作ってくれる。無料。」

---

## 1. このサービスで獲得する層

### まず前提：私たちの事業の現状

私たちは「アスミツ適職診断」というサービスをSNS（TikTok/Instagram/Threads）で運営している。SNSで「アスミツ」と検索するよう訴求し、Google検索経由でサービスに流入させるモデル。マネタイズは、面談をしてからエージェントに送客し、転職が決まったら成功報酬を受け取る形。

実績は以下の通り。

| 指標 | 数値 | 補足 |
|------|------|------|
| PV | 158,905 | SNS→Google検索→LP |
| LINE流入 | 35,349（対PV 22.25%） | LPからLINE登録。CPA約300円 |
| 面談設定 | 2,352（対LINE 6.65%） | LINE登録者のうち面談予約した人 |
| 面談実施 | 1,878（対設定 79.8%） | 予約して実際に面談した人 |
| エージェント送客 | 1,018（対面談 54.2%） | 面談した人のうちエージェントに紹介した人 |
| 年間売上 | 約2,980万円 | エージェントからの成功報酬 |
| 1リスト単価 | 29,268円 | 年間売上 ÷ 送客数 |

**集客力は強い。** CPA300円で35,000人をLINE登録させているのは、SNSマーケとしてかなり優秀。適職診断の競合はリクルート・マイナビ・doda・ミイダスなど大手含め多数いるが、それでもこの数字を出せている。コンセプトとSNSの見せ方が強みだということがわかる。

**しかし、マネタイズが弱い。** 35,349人のLINE登録に対して、実際にマネタイズできているのは1,018人（2.88%）。残り93%、約33,000人のリストが寝ている。年間売上3,000万円は、この集客力に対して明らかに少ない。

### なぜ93%がマネタイズできていないのか

原因は、**収益導線が「面談→エージェント送客」の1本しかない**こと。

この導線で収益化できるのは「今すぐ転職したい人」だけ。しかし適職診断をSNSで受ける人の大半は「今すぐ転職したい」わけではない。

ファネルの数字を見れば一目瞭然：

```
LINE登録: 35,349人（100%）
    ↓ ここで93%が離脱
面談設定:  2,352人（6.65%）
    ↓ ここは問題ない（79.8%が実施）
面談実施:  1,878人
    ↓ ここも問題ない（54.2%が送客）
送客:      1,018人（全体の2.88%）
```

**ボトルネックは明確に「LINE登録→面談設定」の6.65%。** 面談設定以降の転換率は悪くない（面談→送客54.2%）。問題は、**LINE登録した人の93%が面談を設定しない**こと。

なぜ面談しないのか？答えはシンプルで、**まだ転職する気がないから**。「なんとなくキャリアが気になる」「自分に合う仕事って何だろう」という軽い気持ちで診断を受けただけ。面談は重い。

ここで重要なのが**「潜在層」と「顕在層」の違い**。

### 潜在層と顕在層

キャリアに関心がある人は、大きく2つに分けられる。

| | 潜在層 | 顕在層 |
|--|--------|--------|
| 状態 | まだ動いていない | もう動いている or 動こうとしている |
| 気持ち | 「なんとなく気になる」「自分に合う仕事って何だろう」 | 「転職活動中」「書類出してる」「そろそろ転職しようかな」 |
| 温度感 | 低い | 高い |
| 行動 | 診断をやる。情報を見る。でも動かない | 履歴書を書く。求人を探す。面接を受ける |
| エージェント紹介への反応 | 「まだいいかな…」「面談はちょっと…」 | 「相談したい」「求人紹介してほしい」 |
| 人数 | 多い（SNSで大量に集まる） | 少ない（でもマネタイズしやすい） |

**適職診断は「潜在層」を集めるサービス。** 「自分に合う仕事って何？」という好奇心で来る人たち。SNSの短尺動画で「やってみよう」と思わせるのが得意。だから大量に人が来る。

しかし大量に来ても、93%が面談に至らない。それは当然で、**潜在層はまだ転職する気がない**。診断結果を見て「へー」で終わる人が大半。これは適職診断が悪いのではなく、**潜在層を集めるサービスの構造的な限界**。

一方で、**「顕在層」——つまり既に転職活動をしている人や、これから始めようとしている人——を私たちは取れていない。** この層は温度感が高く、エージェント紹介も受け入れやすい。しかし適職診断の「自分を知ろう」という入口では、この層は来ない。なぜなら、この層が欲しいのは「自分を知ること」ではなく「転職活動を前に進めること」だから。

### だから、顕在層を取るための別の入口が必要

適職診断をいくら伸ばしても、取れるのは潜在層。LINE登録者は増えるが、マネタイズ率2.88%の構造は変わらない。

**顕在層を取るには、顕在層が「使いたい」と思う別のサービスが必要。** 潜在層向けの入口（適職診断）と、顕在層向けの入口（新サービス）の2つを持つことで、キャリア領域の全レイヤーをカバーできる。

```
潜在層:  「キャリア気になる」→ 適職診断 → 一部が面談・送客
顕在層:  「転職活動で困ってる」→ 新サービス → 送客（転換率が高い）
```

では、顕在層の中で私たちが取れるのは具体的にどんな人か。

### 顕在層の中で、私たちが獲得できるのは誰か

顕在層（転職活動中の人）は世の中にたくさんいる。大卒のエンジニアも、MBA持ちの経営企画も、高卒の工場勤務も、みんな「顕在層」になりうる。

その中で私たちが獲得できるのは、**SNSマーケの強みが活きる層**。適職診断で既に実績がある層がそのまま対象になる。

#### 属性

- 年収300万円台
- 高卒・専門卒
- 20代〜30代前半
- SNSが情報源（TikTok/Instagram/Threads）
- 初めての転職、または転職経験が少ない

#### 具体的な職種

工場勤務、販売職（アパレル・携帯ショップ等）、飲食（ホール・キッチン）、介護、コールセンター、事務、ドライバー、建設現場など。いわゆる「大手の転職サイトがメインで狙わない層」。

この人たちは大卒でホワイトカラーのキャリアを積んできた人とは、転職活動の事情が全く違う。同じ「転職」でも、やっていることが別物。

#### 大卒ホワイトカラーとの違い

| | 大卒ホワイトカラー | この層（高卒/専門卒・年収300万台） |
|--|------------------|-------------------------------|
| 転職期間 | 3〜6ヶ月かけて慎重に | 2週間〜1ヶ月。決まる時は一瞬 |
| 選考プロセス | 書類→1次面接→2次面接→最終 | 書類→面接1回→内定。即日内定もある |
| 重視されること | スキル・実績・職歴の一貫性 | 人柄・やる気・すぐ来れるか |
| 履歴書の書き方 | ある程度知っている or エージェントが添削してくれる | **わからない。教えてくれる人もいない** |
| 職務経歴書 | 書く（転職活動の基本） | **存在すら知らない人も多い** |
| 使うデバイス | PC | **ほぼスマホだけ。PCを持っていない人も** |
| 転職の情報源 | 転職サイト・エージェント・LinkedIn | **SNS（TikTok/Instagram）** |
| エージェントとの関係 | 普通に使う。複数社に登録 | **相手にされない感覚がある。そもそも使い方がわからない** |

求人広告でよく見る「2週間で内定」という訴求が刺さる層でもある。この層の転職はスピードが速い。書類→面接1回→内定が普通で、長々と選考が続くことは少ない。だからこそ、**履歴書さえ出せれば話は早い**。履歴書がボトルネックになって動けないのがもったいない。

#### この層が置かれている状況

**「エージェントに相手にされない」感覚がある。** ビズリーチやリクルートダイレクトスカウトは年収600万以上がターゲット。ハイクラス転職のCMを見ても「自分には関係ない」と感じている。実際に、年収300万台向けのスカウトサービスは日本に1つも存在しない（詳細はmarket-research参照）。

低年収層向けのエージェント（ハタラクティブ、UZUZ、DYM就職等）は存在するが、自分で見つけられていない人が多い。そもそも「転職エージェント」という概念自体がピンと来ていない人もいる。

**SNSが情報源。** 転職について調べるとき、Google検索より先にTikTokやInstagramで情報を探す。「転職 やり方」ではなく、TikTokで「転職してみた」「仕事辞めた」みたいな動画を見て情報収集している。私たちの適職診断がSNS経由でCPA300円・35,000人を集められているのは、この層の行動パターンにマッチしているから。

**日本の給与所得者の51.1%が年収400万円以下。** この層は「ニッチ」ではなく、日本の労働者の半分以上を占めるボリュームゾーン。にもかかわらず、この層向けのキャリアサービスは圧倒的に手薄。大手転職サイトもエージェントもスカウトサービスも、基本的にハイクラスに集中している。

### この人たちが転職を始める時、何につまずくか

この層が「転職しよう」と動き出した時に、**履歴書がボトルネックになっている**。具体的には2つの段階でつまずいている。

#### つまずき①：これから転職する人 —「履歴書がそもそも書けない・めんどくさい」

日本の履歴書はJIS規格でフォーマットが決まっていて、基本的には穴埋め作業。名前、住所、学歴、職歴は欄に書けばいい。

**しかし「志望動機」と「自己PR」だけは違う。** この2つだけは「自分で考えて書く」必要がある自由記述欄。

高卒・専門卒で転職経験が少ない人にとって、ここが最大のハードルになっている。なぜなら：

- **志望動機を真剣に書いた経験がない。** バイトの面接で履歴書を出したことはあっても、「御社の〇〇に惹かれ…」みたいな志望動機を書いたことがない
- **自己PRの概念がよくわからない。** 「自分の強み」と言われても何を書けばいいかわからない。「真面目です」「頑張ります」以外に思いつかない
- **ネットで調べても参考にならない。** 「志望動機 書き方」でググると出てくる例文は「前職でのマネジメント経験を活かし」「貴社のグローバル展開に」みたいな大卒ハイスペ向け。自分の経歴では使えない
- **周りに聞ける人がいない。** 大卒なら大学のキャリアセンターやOB/OG、転職エージェントに添削してもらえる。この層にはそういうサポートがない

加えて、**そもそも履歴書を書くこと自体がめんどくさい**。スマホネイティブの20代にとって、紙のフォーマットに手書きしたり、PCで入力してPDF化するプロセスは「だるい」。

心理の順番：

```
「転職したいな」
  ↓
「でも履歴書書かないと…」
  ↓
「志望動機？自己PR？何書けばいいの？」（わからない）
  ↓
「ググったけど参考にならん」（不安）
  ↓
「もういいや、めんどくさい」（諦め）
  ↓
  転職活動が始まらない
```

**「書けない」と「めんどくさい」の両方が壁になって、結果として転職活動が始められない。** 転職したい気持ちはあるのに、履歴書で止まっている人がたくさんいる。

#### つまずき②：転職活動中の人 —「書類が全然通らない」

一方、既に転職活動を始めて書類を出している人の中にも、困っている人がいる。

```
「何社も出してるのに書類で全部落ちる」
「どこが悪いのかわからない」
「面接にすらたどり着けない」
「もう何社落ちたかわからない。心が折れそう」
```

この人たちの多くは、**履歴書の志望動機と自己PRの書き方に問題があることに気づいていない**。最初になんとなく書いたものをそのまま使い回している。全ての応募先に同じ志望動機を送っている人もいる。

「書類で落ちる原因」として本人が考えるのは「学歴が低いから」「経験が足りないから」。**書き方を変えれば通る可能性があることを知らない。**

このサービスでAIが志望動機・自己PRを作り直してくれたら、2つの効果がある：

1. **「もしかして書き方が悪かったのかも」と気づかせることができる**
2. **実際に通過率が上がれば、それ自体が最強のSNS訴求になる**（「これ使ったら書類通った」）

#### 2つのつまずきに1つのサービスで対応できる

| | これから転職する人 | 転職活動中の人 |
|--|------------------|-------------|
| 状態 | 履歴書が書けなくて動けない | 書類を出してるが通らない |
| 悩みの核心 | 書き方がわからない・めんどくさい | どこが悪いかわからない |
| このサービスの価値 | AIが書いてくれるから始められる | AIで作り直したら通過率が上がる |
| SNSでの訴求 | 「履歴書の書き方わからない人、AIが全部作ってくれるよ」 | 「これ使ったら書類の通過率上がった」 |
| エージェント送客 | 「履歴書できた→次は応募先」で送客 | 「書類通った→もっといい求人ない？」で送客 |

**同じサービスだが、SNSの訴求を変えるだけで両方の層にリーチできる。** そしてどちらの層も、エージェント送客（マネタイズ）に繋がりやすい。

### なぜこの層を獲得しにいくのか

ここまで読むと「なぜこの層なのか」はある程度見えてきたと思うが、改めて3つの理由を整理する。

**理由1: 私たちのSNSマーケで最もリーチしやすい層だから。**

アスミツ適職診断のユーザーも年収300万台・高卒/専門卒が多い。TikTok/Instagram/Threadsでこの層にリーチできることは適職診断で証明済み。具体的には、CPA300円で35,000人をLINE登録させた実績がある。

適職診断の競合はリクルート・マイナビ・doda・ミイダスなど大手を含めて多数存在する。しかしそれでもSNSマーケのコンセプトと見せ方で今の数字を出せている。これは「競合がいるかどうか」より「SNSでの見せ方」が重要だということの証明。履歴書サービスでも同じ手法で再現できる。

**理由2: この層は転職エージェントに送客しやすいから（=マネタイズしやすい）。**

ここが適職診断との決定的な違い。

適職診断に来る人は「なんとなく気になる」潜在層。面談を提案しても「まだいいかな」となる（LINE→面談設定率6.65%）。

一方、この新サービスに来る人は「転職活動中で書類に苦戦している」or「これから転職しようとしていて履歴書を準備している」顕在層。**まだ転職先が決まっていない**から、「あなたの経歴に合う求人がありますよ」と提案したら受け入れやすい。

送客の転換率は適職診断の6.65%より高くなるはず。なぜなら、このサービスに来る人は既に「転職する」という意思決定をしている（or しかけている）から。適職診断の「なんとなく興味ある」とは温度感が全く違う。

**理由3: この層向けのサービスが空白だから。**

履歴書の書き方に困っている高卒/専門卒向けに、AIで志望動機・自己PRを無料で作ってくれる独立したサービスは存在しない。

**マイナビ・doda・リクナビ**もAI志望動機生成機能を持っているが、全て自社転職サイトの会員限定機能。「マイナビに登録しないと使えない」「dodaに登録しないと使えない」。会員登録の手間がハードルになる上、登録すると大量のメールや電話が来ることを知っている人は避ける。「会員登録なしで誰でもすぐ使える独立したサービス」がない。

**yagish**（累計1,000万人の履歴書作成サービス）もAI志望動機を提供しているが、月額550円の有料機能。しかもこの有料プランは苦戦して2025年11月に一部終了している。つまり**需要はあるがyagishは課金壁で取りこぼしている**。

私たちは最初から無料で提供する。yagishが550円で取りこぼしている需要を、無料で全部拾う。

### この層をどう獲得するか（訴求の設計）

獲得する層が2つあるので、SNSの訴求も2軸で設計する。

#### 訴求の2軸

| | 層A：これから転職する人 | 層B：転職活動中の人 |
|--|----------------------|-------------------|
| 状態 | 履歴書が書けなくて動けない | 書類を出してるが通らない |
| 悩み | 「書き方わからない」「めんどくさい」 | 「何社出しても全部落ちる」 |
| 刺さる訴求 | **「AIが書いてくれるよ」** | **「これ使ったら通過率上がった」** |
| 訴求の性質 | 不安の解消（助けてくれるものがある） | 成果の提示（実際に結果が出た） |

さらに、**両方の層に刺さる訴求がもう1つある**。

この層の転職は「2週間で内定」が普通にあり得るスピード感。書類→面接1回→内定。つまり**履歴書さえできれば、すぐ内定まで行ける**。「このサービス使ったおかげで2週間で内定出た」は、層Aにも層Bにも刺さる最強の成果訴求。

| | 層A | 層B | 共通 |
|--|-----|-----|------|
| 訴求 | AIが書いてくれる | 通過率が上がった | **2週間で内定出た** |
| 心理 | 「始められそう」 | 「やり直してみよう」 | **「自分もいけるかも」** |

#### 層A向け：「AIが書いてくれる」訴求

この層は「書き方がわからない」「自信がない」が出発点。だから**「AIが代わりにやってくれる」という安心感**が最も刺さる。

SNSでの具体的な見せ方：

```
「正直、履歴書の書き方全然わからんかった」（共感）
 ↓
「でもこのアプリ使ったらAIが志望動機も自己PRも全部出してくれた」（解決）
 ↓
「しかも無料」（驚き）
```

訴求のバリエーション：
- 「履歴書の書き方わからない人、AIが全部作ってくれるよ」
- 「自己PRとか何書けばいいか意味不明だったけど、AIに聞いたら3秒で出てきた」
- 「高卒で自己PR何も思いつかなかったけど、AIが出してきた文章が普通にすごかった」
- 「志望動機の書き方ググるのやめて、AIに作ってもらった方が早かった」
- 「志望動機も自己PRも30秒で終わった。もう自分で書かなくていい」

#### 層B向け：「通過率が上がった」訴求

この層は既に書類を出しているが落ち続けている。**「これを使ったら結果が変わった」という成果**が最も刺さる。

SNSでの具体的な見せ方：

```
「書類で落ちまくってた」（共感）
 ↓
「履歴書のAI使って志望動機と自己PR作り直した」（行動）
 ↓
「そしたら書類通るようになった」（成果）
```

訴求のバリエーション：
- 「これ使ったら書類通過率が上がった。だからみんな使った方がいいよ」
- 「書類で落ちまくってたけどこれ使ってから通るようになった」
- 「書類落ちる原因、志望動機と自己PRだったかもしれない」
- 「このサービス使ったおかげで2週間で内定出た」

#### 2つの訴求のSNS運用

同じサービスを、**SNSの投稿ごとに訴求を切り替えて発信する**。

```
月曜: 層A向け「履歴書の書き方わからない人、AIが作ってくれるよ」
水曜: 層B向け「書類落ちまくってた人がこれ使ったら通った話」
金曜: 層A向け「自己PR30秒で終わった」
```

**同じサービスへの導線だが、入口の訴求を変えることで、異なる悩みを持つ2つの層を獲得できる。** これは適職診断ではできなかったこと（適職診断は潜在層向けの訴求しかできない）。

#### SNSの動画フォーマット

どちらの訴求でも、基本のストーリー構成は同じ：

```
① 共感（こういう状況だった）
 ↓
② 解決（このサービスを使った）
 ↓
③ 結果（こうなった）
 ↓
④ CTA（「アスミツ履歴書」で検索）
```

適職診断と同じ導線で、SNSの動画内で**「アスミツ履歴書」でGoogle検索**と訴求し、検索させてサービスに流入させる。

#### 訴求の前提にあるリスクと、検証で決めること

上記の訴求は「志望動機・自己PRが書けなくて困っている」ことを前提にしている。しかし、**この前提が正しくない可能性がある。**

この層（飲食・工場・介護・販売等）の転職は：
- **志望動機を空欄で出しても面接に呼ばれる**ことがある（人手不足だから）
- **職務経歴書がそもそも不要**な求人が多い
- 書類の「質」よりも「面接での人柄」で決まる世界

つまり、**志望動機をAIで書いてあげることの価値が、想定より低い可能性がある。**

その場合、本当の価値は「志望動機のAI生成」ではなく**「履歴書作成の手間自体をゼロにすること」**かもしれない。

```
仮説A: この層は志望動機が書けなくて困っている → AIが書いてくれることに価値がある
仮説B: この層は履歴書を作ること自体がだるい → 3分で完成することに価値がある
```

**どちらが正しいかは4/11のLINEテストで検証する。**

| 見るべきデータ | 仮説Aが正しい場合 | 仮説Bが正しい場合 |
|-------------|----------------|----------------|
| AI志望動機の利用率 | 高い（ほとんどの人が使う） | 低い（スルーして穴埋めだけで完了） |
| AI自己PRの利用率 | 高い | 低い |
| 志望動機欄の入力状況 | AIで生成したものを採用 | 空欄のまま or 自分で短く書く |

**テスト結果に応じて訴求の優先順位をシフトする：**

| テスト結果 | メイン訴求 | サブ訴求 |
|-----------|----------|---------|
| AI利用率が高い | 「AIが志望動機書いてくれる」 | 「スマホで3分で完成」 |
| AI利用率が低い | 「スマホで3分で履歴書完成。無料」 | 「志望動機もAIで作れる」 |

ただし、AI利用率が低い場合でも**yagishとの差別化のためにAI機能は残す**。無料で使えるAI志望動機は、yagishの有料機能（月550円）に対する明確なアドバンテージになる。訴求の重心が変わるだけで、サービスの機能は変えない。

## 2. 何を提供するか

### サービスの中身

AIが志望動機と自己PRを自動生成してくれる、無料の履歴書作成サービス。

#### なぜ「履歴書」なのか

海外の「Resume」と日本の「履歴書」は全く別物。海外のResumeはフォーマット自由で、書き方やデザインで差がつく。だからAIで「最適化」する意味がある。

しかし日本の履歴書はJIS規格でフォーマットが決まっている。名前、住所、学歴、職歴——書く欄が固定されていて、ツールとしての差別化がほぼできない。yagishも含め、日本の履歴書ツールは基本的に「穴埋めのデジタル化」でしかない。

**ただし「志望動機」と「自己PR」だけは自由記述。** ここだけがAIの価値を出せるポイントであり、第1章で述べた通りこの層が最もつまずいている箇所。つまり「履歴書ツール」として出しつつ、**本当の価値は「AIによる志望動機・自己PRの自動生成」にある。**

また、「職務経歴書」ではなく「履歴書」を入口にする理由がある。この層にとって職務経歴書は馴染みがなく、存在すら知らない人も多い。一方、履歴書はバイトの面接でも書いた経験がある。**入口のハードルを下げるために、誰でも知っている「履歴書」を前面に出す。**

#### 機能一覧

| 機能 | 内容 | なぜ必要か |
|------|------|----------|
| 履歴書作成 | JIS規格の穴埋め式。基本情報（名前・学歴・職歴）を入力 | 最低限の土台。yagishと同じ |
| **AI志望動機生成** | 職種・経歴を入力するだけで、AIが志望動機を3パターン提案。選ぶだけ | **このサービスの核心。** この層が最も困っている「何を書けばいいかわからない」を解決する |
| **AI自己PR生成** | 同上。AIが自己PRを3パターン提案。選ぶだけ | 志望動機と並ぶもう1つのハードル。「自分の強み」を言語化できない人のためのもの |
| PDF保存 | 完成した履歴書をPDFでダウンロード | スマホで作ってそのまま送れる |
| コンビニ印刷 | プリンターがなくてもコンビニで印刷できる | この層はPCもプリンターも持っていないことが多い。yagishもこの機能で伸びた |
| 料金 | **全機能完全無料** | yagishのAI志望動機は月550円（苦戦して一部終了）。私たちは無料で提供してyagishが取りこぼしている需要を拾う |

#### なぜ全部無料にできるのか

AI生成のコスト（詳細は第7章）：
- 志望動機1件あたり約0.06〜0.08円（軽量モデル利用時）
- 月10万件生成しても6,000〜8,000円

このコストはエージェント送客の成功報酬（1件あたり数万円）で十分にペイする。ユーザーからお金を取る必要がそもそもない。

### 既存サービスとの違い

AI志望動機生成は新しい技術ではない。既に複数のサービスが提供している。だから「AIで作れます」だけでは差別化にならない。**差別化は「誰に」「どうやって届けるか」にある。**

#### vs yagish（累計1,000万人の履歴書作成サービス）

yagishは日本最大の独立型履歴書サービス。年商約5億円、ユーザー250万人。最も直接的な比較対象。

| | yagish | アスミツ履歴書 |
|--|--------|-------------|
| ユーザーの来る理由 | 「履歴書が必要だから」（ツール目的） | 「書き方わからないから助けてほしい」（不安の解決） |
| 訴求 | 「楽に作れる」（利便性） | 「AIが書いてくれる。書類通る。2週間で内定」（不安の解消＋成果） |
| 集客チャネル | SEO（100%検索流入。SNSは一切やっていない） | SNS（TikTok/Instagram/Threads） |
| AI志望動機 | 月550円の有料機能（苦戦して2025年11月に一部終了） | **完全無料** |
| プライバシー | 不透明。登録後にエージェントから大量の電話勧誘が来るとの不満が多数（App Store低評価の主因） | ユーザーが自分で公開範囲を選べる透明な設計 |
| ユーザーの状態 | 応募先が既に決まっていることが多い → 送客しにくい | まだ決まっていない → 送客しやすい |

**yagishとは集客チャネルが被らない**（SEO vs SNS）ので、同じ構造でも共存できる。正面からぶつかる必要はない。

#### vs マイナビ/doda/リクナビ（大手転職サイト）

マイナビ（AI Pencil）、doda（自己PR発掘診断）、リクナビNEXT（AI作成補助）も既にAI志望動機生成を無料で提供している。しかし**全て自社転職サイトの会員限定機能**。

| | 大手転職サイト | アスミツ履歴書 |
|--|-------------|-------------|
| 利用条件 | マイナビ/doda等に会員登録が必要 | **会員登録なしでも使える** |
| 位置づけ | 転職サイトの付随機能の1つ | **独立したサービス** |
| 集客 | 自社サイトの既存会員向け | **SNSで直接バズらせる** |
| 会員登録のハードル | 登録すると大量のメール・電話が来ることを知っている人は避ける | 段階的な情報取得。最初は個人情報なしで使える |

yagishが成功した構図と全く同じ。リクナビもdodaも履歴書ツールを持っていたが、yagishは「独立した履歴書サービス」として出して1,000万人集めた。**大手の付随機能 vs 独立サービス**の構図で、独立サービスが勝ったケース。AI志望動機でも同じことが起きる可能性がある。

## 3. どうやって届けるか

SNSでの訴求内容は第1章「この層をどう獲得するか」で定義済み。ここでは**訴求を見た人がサービスを使って、マネタイズに至るまでの仕組み**を説明する。

### SNSからサービスへの導線

適職診断と同じ方式。SNSの動画内で**「アスミツ履歴書」でGoogle検索**と訴求し、検索させてサービスに流入させる。

なぜ直接URLではなくGoogle検索なのか：
- SNSの短尺動画でURLを打ち込む人はいない
- 「〇〇で検索」は適職診断で実績がある導線
- Google検索結果の1位を取ることで、SEOからの自然流入も後から取れるようになる

### ユーザー体験フロー

SNSで動画を見た人が、実際にサービスを使って、マネタイズ（送客）に至るまでの一連の流れ。

```
① SNSで訴求を見る
  「履歴書の書き方わからない人、AIが全部作ってくれるよ」
  「アスミツ履歴書で検索」
    ↓
② Google検索 → サービスにアクセス
    ↓
③ まず試す（個人情報なし）
  職種と経歴だけ入力 → AIが志望動機のサンプルを生成
  「おっ、ちゃんとしたの出てきた」→ 続けるモチベーションが生まれる
    ↓
④ 履歴書を作る
  基本情報を穴埋め（名前・学歴・職歴）
  志望動機 → AIが3パターン提案 → 選ぶだけ
  自己PR → AIが3パターン提案 → 選ぶだけ
    ↓
⑤ 完成 → 保存/出力
  PDF保存 or コンビニ印刷
  ※保存にはメールアドレスで無料アカウント作成
    ↓
⑥ 送客ポイント
  ポップアップ:「あなたの経歴から条件が良くなる求人サンプル」
  求人サンプルを表示
  「気になる」→ タップ → エージェント送客
```

ポイントは**③の「まず試す」ステップ**。いきなり個人情報を求めず、AIの価値を先に体験してもらう。「これ使える」と思ってから情報を入力してもらう設計。

### 個人情報の取得は段階的に行う

SNS経由の初見ユーザーにいきなり個人情報を求めると「怪しいサービスなのかな」と思われて離脱する。特にこの層はSNSで見つけたサービスに対して警戒心が強い。

だから**段階的に情報を取得する設計**にする。

| ステップ | 何ができるか | 必要な情報 | ユーザーの心理 |
|---------|------------|-----------|-------------|
| Step 1 | AIの志望動機生成を試す | 個人情報なし（職種・経歴だけ） | 「試してみよう」→「おっ、使えるじゃん」 |
| Step 2 | 履歴書を保存/PDF出力 | 無料アカウント作成（メールアドレス程度） | 「保存したいからアカウント作るか」 |
| Step 3 | スカウトを受け取る | 経歴を詳しく登録 | 「求人紹介してほしいから登録するか」（将来のPhase 2） |

**Step 1で価値を体験させてからStep 2に進むのが重要。** 価値を感じていない状態でアカウント作成を求めると離脱する。

### LINE登録は必須にしない

適職診断は「SNS→LINE登録→結果受け取り」の導線。これは適職診断が「結果を受け取る」コンテンツだから成立する。

履歴書サービスは**ツール**。ツールを使いたい人に「まずLINE登録して」は邪魔。「履歴書作りたいのになんでLINE？」となって離脱する。

サービス上（ブラウザ or アプリ）で直接使える設計にする。yagishもLINE挟んでいない。

ただし、LINEを完全に排除するわけではない。サービス利用後に「新機能のお知らせ」「求人情報」をLINEで受け取るオプションとして提供するのはあり。**必須ステップにしない、というだけ。**

## 4. どうやって稼ぐか

ユーザーからはお金を取らない。全機能完全無料。ではどうやって収益化するか。

### Phase 1（初期）: エージェント送客

**仕組み：** 履歴書を作った人に転職エージェントを紹介し、成功報酬を受け取る。適職診断で既にエージェント送客の仕組みがあるので、それを流用できる。

**送客の導線：**

```
履歴書完成！
  ↓
ポップアップ:「あなたの経歴から条件が良くなる求人サンプル」
  ↓
求人サンプルを表示（実際の求人に近いもの）
  ↓
「気になる」→ タップ → エージェント送客
```

**なぜこのタイミングか：** 履歴書が完成した直後は達成感がある。「書類ができた、次は応募先」という気持ちになっている。このタイミングで「あなたの経歴に合う求人がありますよ」と出せば、自然な流れでエージェント紹介に繋がる。押し売り感が少ない。

**yagishより送客の転換率が高くなる理由：**

yagishに来る人は「履歴書が必要だから」来ている。多くの場合、応募先は既に決まっている。スカウトやエージェント紹介は「間に合ってます」になりやすい。yagishのヤギオファー（企業スカウト）のスカウト承認率が公開されていないのは、おそらくここが弱いから。

一方、私たちのユーザーは2パターンあるが、どちらも送客しやすい：

| ユーザー | なぜ送客しやすいか |
|---------|-----------------|
| これから転職する人 | 履歴書ができた→「次はどこに応募しよう」→ エージェント紹介が自然 |
| 転職活動中の人 | 書類が通るようになった→「もっといい求人ないかな」→ エージェント紹介が自然 |

**共通して「まだ転職先が決まっていない」のが重要。** だからエージェント紹介を受け入れやすい。

### Phase 2（将来）: B2Bスカウト

ユーザー数が十分に集まったら、企業向けにスカウト機能を開放する。yagishのヤギオファー（月額1万円・6,000社導入）と同じ構造。

年収300万台向けのスカウトサービスは日本に存在しないので（第1章参照）、この市場を最初に取れる可能性がある。

ただし、Phase 2はユーザーを数万人規模で集めてからの話。今の段階で詳細は詰めない。まずはPhase 1（送客）でユーザーを集めて収益を出すことに集中する。

## 5. 既存事業（適職診断）との関係

### カニバらない。補完関係にある

よくある懸念：「同じ会社が似たサービスを2つ出したら、ユーザーを食い合うのでは？」

答えはNo。**入口の動機が全く違う**から。

| | 適職診断 | アスミツ履歴書 |
|--|---------|-------------|
| 取る層 | 潜在層（まだ動いてない人） | 顕在層（もう動いてる or 動こうとしている人） |
| 入口の動機 | 「自分に合う仕事って何？」 | 「履歴書の書き方がわからない」「書類が通らない」 |
| ユーザーの温度感 | 低い（なんとなく気になる） | 高い（転職活動中 or 直前） |
| SNSで刺さる訴求 | 「あなたに合う仕事がわかる」 | 「AIが履歴書書いてくれる」「書類通過率上がった」 |
| マネタイズ | 面談→送客（転換率6.65%） | 履歴書完成→送客（転換率はこれより高いはず） |
| 収益導線 | LINE登録→面談→送客 | サービス利用→送客 |

適職診断は「自分を知りたい」人が来る。アスミツ履歴書は「書類を作りたい」人が来る。同じ人が両方使うことはあっても、**それぞれ別の目的で使うので食い合わない**。

### 両方持つメリット

```
潜在層:  「キャリア気になる」→ 適職診断
顕在層:  「転職活動で困ってる」→ アスミツ履歴書
```

**キャリア領域の「まだ動いていない人」から「もう動いている人」まで全カバーできる。** 片方しかないと、片方の層しか取れない。両方あることでキャリア事業全体のパイが広がる。

さらに、**潜在層→顕在層への移行も取れる**。適職診断で「自分に合う仕事がわかった」人が、次のステップとして「じゃあ履歴書作ろう」となった時に、アスミツ履歴書に来る。今まではこの移行先がなくてユーザーが離脱していた。

### 既存LINEリスト（33,000人）の活用

Webサイトが完成してから、既存のLINEリストにテスト配信する。

33,000人の中には「適職診断はやったけど転職活動には至っていない」人が大量にいる。この人たちの一部は、**その後転職活動を始めている or 始めようとしている可能性がある**。「履歴書もAIで作れるよ」と案内することで、眠っていたリストから新サービスの初速を出せる。

これは**適職診断の93%の「寝ているリスト」を別のサービスで掘り起こす**ことでもある。適職診断では面談に至らなかった人でも、「AI履歴書が無料で作れる」なら使ってくれるかもしれない。そこから送客に繋がれば、今まで収益化できなかったリストが活きる。

## 6. なぜこのサービスをやるのか（まとめ）

ここまでの内容を一言でまとめると：

**「適職診断で取れなかった顕在層を、AI履歴書サービスで獲得し、新しい収益導線を作る」**

| 課題 | 解決策 |
|------|--------|
| 適職診断のマネタイズ率が2.88%と低い | 顕在層を取る別の入口を作り、送客転換率を上げる |
| 93%のLINEリストが寝ている | 別のサービスで掘り起こす |
| 収益導線が1本しかない | 2本目の導線を作る |
| 潜在層しか取れていない | 顕在層向けのサービスで両方カバーする |

適職診断（潜在層向け）とアスミツ履歴書（顕在層向け）の2つを持つことで、キャリア事業全体の収益基盤が強くなる。

## 7. 実現可能性

「本当にできるのか」「コストは大丈夫か」という疑問に答える。

### AI生成のコスト

AIで志望動機・自己PRを生成するコストは、驚くほど安い。

| モデル | 志望動機1件あたり | 月10万件生成した場合 |
|--------|----------------|-------------------|
| Gemini 2.0 Flash（最安） | 約0.06円 | 約6,000円 |
| GPT-4o-mini | 約0.08円 | 約8,000円 |
| GPT-4o（高品質） | 約1.4円 | 約14万円 |

**月10万件生成しても6,000〜8,000円。** エージェント送客が1件成功するだけで数万円の成功報酬が入ることを考えると、AI生成コストは誤差。完全無料で提供しても全く問題ない。

yagishがAI志望動機を月550円で課金していたのは、コスト回収のためではなく収益化のため。そして苦戦した。私たちは最初から無料にして、送客で回収する設計。

### 技術的な実現性

- **開発コストは低い。** LLM（大規模言語モデル）のAPIを呼ぶだけで基本的に動く。一からAIを開発する必要はない
- **プロンプト設計は確立されている。** 「職種・経歴を入力→志望動機を出力する」プロンプトは、既に多くのサービスで使われている技術
- **前例が多い。** マイナビ（AI Pencil）・doda・リクナビ・HelloBoss・就活AI等が既に同様の機能を実現済み。技術的なリスクはほぼない
- **日本の履歴書はフォーマット固定。** JIS規格に沿った穴埋め式なので、履歴書作成ツール自体の開発も複雑ではない

### AI品質

- リリース前に実際の職種・経歴パターンでテストして品質を確認する
- **「自分で書くよりマシ」レベルで十分。** ハイレベルな文章は求めない
- なぜなら、この層の現状は「何も書けない」or「真面目です、頑張りますしか書けない」。**AIが出すだけで、現状から大幅に改善される**
- テスト時に確認すべき点：職種ごとの精度差、敬語の自然さ、情報が少ない場合の出力品質
