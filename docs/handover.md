## プロジェクトの全体像（1分でわかる版）

```
課題: 適職診断のマネタイズ率が2.88%。93%のリストが寝ている
原因: 潜在層しか取れていない。顕在層（転職活動中の人）を取れていない
解決: 顕在層向けの新サービスを2つ作る

サービス①: アスミツ履歴書
  → AIが志望動機・自己PRを無料で自動生成する履歴書サービス
  → SNSで集客 → エージェント送客でマネタイズ

サービス②: スカウトサービス（将来）
  → ユーザーが増えたら企業にスカウト機能を月額で売る
  → CPA300円の集客力で、他社が成り立たない年収300万台市場で成立する
```

## 4/11 LINEテストの設計

3,500人を1,750人ずつに分割してA/Bテスト。

| グループ | 人数 | 配信文言 | 検証すること |
|---------|------|---------|------------|
| A | 1,750人 | 「AIが志望動機書いてくれる」 | AI機能に価値があるか |
| C | 1,750人 | 「あなた限定の求人サンプルが届く」 | スカウト訴求が刺さるか + マネタイズが自然か |

### テスト後の判断

| 結果 | 次のアクション |
|------|-------------|
| Aの遷移率が高い | 「AI志望動機」をメイン訴求にしてSNS開始 |
| Cの遷移率が高い | 「求人紹介」をメイン訴求にしてSNS開始。スカウトサービスの優先度を上げる |
| 両方低い | 配信文言を変えて2回目のテスト（別の3,500人） |

## 重要な意思決定の経緯（なぜこうなったか）

### なぜ「履歴書」なのか
- 海外のResumeと日本の履歴書は全く違う（JIS規格で穴埋め）
- 職務経歴書はこの層は知らない人も多い。履歴書は馴染みがある
- 入口のハードルを下げるために「履歴書」を前面に出す

### なぜAI志望動機なのか
- JIS規格の履歴書で唯一「自分で考えて書く」必要があるのが志望動機と自己PR
- yagishはAI志望動機を月550円で提供→苦戦して一部終了。需要はあるが課金壁で取りこぼし
- 私たちは無料で提供。コストは月10万件で6,000〜8,000円

### なぜスカウト訴求（パターンC）を試すのか
- この層は「どこに応募すればいいかわからない」が最大の悩み
- 「あなた限定の求人が届く」は「自分にもスカウトが来るの？」という驚きになる
- さらに、最初から「連絡が来る」と期待しているので電話に出る（マネタイズリスクの解消）

### なぜ年収300万台向けスカウトが成立するのか
- 他社はユーザー1人の獲得に1〜4万円かかる。私たちはCPA300円
- 年収300万台は826万人のボリュームゾーン。需要はある。ビジネスモデルが合わなかっただけ
- ROXX Zキャリアが同じ層で上場している（成立の証明）

次の会話で詰めるべきこと：

| 論点 | 詳細 |
|------|------|
| **35歳以上の扱い** | 今の適職診断は18〜34歳のみ面談。新サービスでは35歳以上も来る。マネタイズ対象にするか |

| **配信文言の具体的な文面** | A/Bテストの配信文をまだ書いていない |
